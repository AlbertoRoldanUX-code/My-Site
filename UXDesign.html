UX is nothing if not a process

Project example:
FLY UX

Overview

My task was to design a new mobile app for a startup airline called Fly UX. My objective was to use great design as a source of competitive advantage over much bigger, but also much more bureaucratic airlines. I focused specifically on the flight booking process: how users can search for, find and select flights on mobile.

User Research

✔ Surveys ✔ Interviews ✔ Usability Tests

Initially, I run surveys to learn more about the goals of the customers. It’s the easiest way to gather user requirements and it augments my Usability Testing efforts. Right before the Usability test, I interview the participant so that I can identify the context.

Analysis

✔ Affinity Diagram ✔ Customer Journey Map

Now it’s time to organize the unstructured data I gathered during the research phase (ideally with a team to build consensus) to clearly articulate the precise problem or set of problems I’m trying to solve.

Design

✔ Flow Diagram ✔ Sketches

Here I define a high-level flow focused on the journey from searching for flights to selecting seats, addressing all the issues in user goals identified in my research and analysis. Later on, I start sketching the screens and screen states building on the flow diagram.

Prototype

Check out the medium-fidelity prototype I created to test the high-level flow, screen layouts, text and basic interactions.

Testing

When conducting additional usability testing with the same tasks on my new design, it showed an increase in findability compared to the previous round of testing. Users were able to complete the overall job quickly, efficiently and smoothly.

Wireframes

At the end of the UX Design cycle is when I’m ready to create the wireframes, with lots of detail and good clear instructions, ready to hand over to developers to build.

What I learned

My design changed as I learned more about how an airline works and the insights I got from the Usability Testing were so valuable. Because of this project, I realized the importance of prototype testing for exploring new design concepts. It made me test new designs to make sure they were viable solutions before putting in the development effort.
